DARS Parser

A University requirements reporter written in Ruby.


The DARS Parser is a simple Ruby application meant to help Portland State University students better visualize their university degree requirements.

Background

DARS, the Degree Auditing Report System, is an official university tool which can be used by students to generate a text-based report of their progress. Unfortunately, this report isn't pretty, and only provides a colorless wall of text. The purpose of this project was to create a very simple method to take the output of DARS and feed it into a visualizer to create images that immediate convey university status.

Implementation

From the beginning, my plan for the DARS parser was to act as a lexer of sorts for the wall of text that is produced by DARS. The concept is based around using regular expressions to capture important pieces, sections, and sub-sections of the report. Luckily, DARS has a (mostly) standard format for how sections are labeled, so I was able to extract most of the useful information.

The project was actually implemented twice - first in Javascript and then in Ruby. I felt the original Javascript implementation wasn't scalable enough for future extensions to the parser, so I rewrote it entirely in Ruby. With the new platform in place, I plan to eventually move away from using regular expressions and build an actual character-by-character parser around the report to better format what the parser can display. It is currently difficult to extract the most meaningful bits from words and sections that could be discarded at no loss of the user, and I believe a more powerful parser is the solution to that problem.
