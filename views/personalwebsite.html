My Website

A small website and blog written using standard web technologies.


This website is a fairly simply constructed project built simply for the fun of it. Much of the styling is achieved using the Bootstrap front-end framework. The blog section utilizes a simple, custom-built PHP based framework sitting on top of a PostgresQL database.

  • The site on GitHub

Technology

  • The primary technologies used to create this website are HTML, CSS, JavaScript, and PHP. A PostgreSQL database is also utilized to serve content for the blog section.

  • In order to maintain a visually appealing and consistent look throughout the site, the Bootstrap front-end framework is used to create structure and styling.

  • The site is hosted through the Maseeh College of Engineering and Computer Science at Portland State University.

Design

In terms of aesthetics, the site is simply intended to look clean and modern. This is mostly accomplished using Bootstrap - particularly the unifying navigation bar.

Probably the most unique aspect of the implementation of this site is how sections of the site are loaded. I wanted to see if I could create a fluid, single page site that didn't require full page reloads for every piece of content. Instead, I experimented with simply using JavaScript to load HTML into predefined sections of the main page without fetching a new URL. The implementation looks like this:

 98     $('#homeButton').on('click', function() { loadPage('home') })
 99     $('#resumeButton').on('click', function() { loadPage('resume') })
100     $('#calendarButton').on('click', function() { loadPage('calendar') })

Of course, this experimental approach has its share of problems. For one thing, refreshing while looking at any piece of content will result in the home page being loaded again. Also, the blog occasionally takes a few moments to load, leaving a blank page while it does. I'm convinced there is a way to do what I wanted to do, but I don't think this is it. Perhaps a Sinatra or Rails implementation would make it simpler.

Blog

Previously, my blog ran on a small PHP framework that I created from scratch to load and display posts from a PostgreSQL database. Though it did work, there were many features that I never got around to implementing, and I found that my approach was probably not going to scale terribly well. To fix this, I removed the PHP framework and began utilizing Jekyll to maintain a somewhat separate, static blog. It still requires a lot of manual labor, but it is certainly working better than my previous framework.

Considerations

In most other circumstances, I would have utilized a Ruby based web framework like Sinatra to create this site. I quite like the ease and flexibility that such frameworks provide, and Ruby is probably my favorite language to write code in.

However, because I'm hosting the site on the College's servers, I'm limited in what technologies I can use. All files must be served on a per user basis out of standard home directories, so running a Sinatra app is out of the question.

This is why I was forced to create a PHP based framework for my blog, where I would have preferred to use Sinatra along with something like SQLite. In fact, as I write this, I realize that a site rewrite using a better framework should probably happen sooner than later!
