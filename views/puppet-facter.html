Puppet and Facter

Open-source configuration management tools writen in Ruby and C++, maintained by Puppet Labs.


Puppet is an open-source IT automation tool that runs on most flavors of Unix, as well as Windows. It allows system administrators to programatically define the state of a machine once, and then simply run Puppet to preserve that state and prevent configuration drift.

Puppet utilizes a tool called Facter to gather information about the systems it runs on. Facter allows Puppet code to be written that takes the differences between various operating systems and platforms into account.

Contributions

As a developer on the open-source platform client team at Puppet Labs, my work primary revolves around these two projects. When I began as an intern in the Summer of 2014, My focus was on adding features for the release of Facter 2.2, which was highlighted by the addition of several high-profile structured facts and various bugfixes.

At the same time, the rewrite of Facter into the new C++ CFacter was underway. I was lucky enough to be able to jump in and help out with development work for the first few releases, and continue to do so as we build up to version 1.0. This included work to ensure parity between the output of Ruby Facter and CFacter - something which is very important, as inconsistent fact values are highly likely to break Puppet manifests for many users.

If not working on Facter, I'm almost certainly working on Puppet. One of the most gratifying aspects of helping with the development of Puppet version 4.0 this past year was ripping out thousands of lines of dead or deprecated code. My team also spent significant time working on the new "all-in-one agent," a large, multi-project update to unify where all the pieces of projects like Puppet end up in the filesystem.
